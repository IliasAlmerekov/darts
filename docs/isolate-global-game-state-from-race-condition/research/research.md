# Research: Isolate Global Game State from Race Condition

**Date:** 2026-03-06
**Ticket:** 6.1 Isolate the global `game-state` store from race conditions between `GamePage` and `GameSummaryPage`
**Task slug:** `isolate-global-game-state-from-race-condition`

## Current State

The shared Nanostore still exposes a single game-state slice for runtime game data, loading, and error state. `GameSummaryPage` keeps its finished standings in local React state, but it still depends on the shared game store for rematch settings and for writing reopened game data back into the runtime flow.
